# 用户手册

Examark 是面向学校考试流程的 OCR/OMR 智能评卷平台。系统覆盖考试创建、试题编写、答题卡生成、扫描识别、阅卷、成绩发布和报告分析，目标是把一次考试从命题到成绩分析连接成可追踪、可审计、可复核的标准流程。

## 适用对象

- **学校管理员与教务人员**：维护学校基础数据、教师、学生、角色权限和考试流程。
- **考官与考试创建人**：配置考试、组织协作人员、发布成绩。
- **试题编写人**：维护试题、题组、试卷信息、预印和印刷数据。
- **扫描人员**：上传扫描件、查看扫描原卷和答题卡绑定信息。
- **阅卷教师**：完成客观复核、主观阅卷和评语录入。
- **AI 教师**：作为匿名阅卷和辅助分析能力的预留角色。

## 核心流程

```text
创建考试
  -> 配置考核班级和考试协作人员
  -> 编写试题和试卷信息
  -> 在线预览并进入预印
  -> 生成试卷 PDF 和答题卡印刷数据
  -> 上传扫描件并完成识别
  -> 客观复核和主观阅卷
  -> 汇总成绩、发布成绩、查看报告
```

## 快速入口

=== "第一次使用"
    从 [系统概览](getting-started/overview.md) 开始，了解页面布局、角色权限和一次考试的完整闭环。

=== "正在组织考试"
    从 [考试全流程](exam-workflow/overview.md) 开始，按步骤完成考试创建、写题、印刷、扫描和阅卷。

=== "配置权限"
    阅读 [权限与匿名化](permissions/overview.md)，明确学校级角色、考试内角色和身份信息能力的区别。

=== "管理员维护"
    阅读 [管理员指南](admin/master-data.md)，维护校历、年级、科目、教师、学生和角色。

## 文档约定

- **入口路径** 指用户在左侧导航或考试工作区中进入功能的位置。
- **适用角色** 指推荐执行该操作的身份，后端仍会按 capability 做最终权限校验。
