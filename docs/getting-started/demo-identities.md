# 演示身份

测试模式下，前端可以通过顶部身份选择器切换演示身份。后端会解析真实 seed 教师、组织、角色、权限和作用域，不把浏览器选择当作生产认证。

## 常用身份

| 身份 | 典型用途 |
| --- | --- |
| 默认管理员 | 全校管理、考试配置、人员配置、调试全部流程 |
| `demo-admin` | 管理员演示账号 |
| `demo-dean` | 教务长，维护教学主数据和考试流程 |
| `demo-homeroom` | 班主任，查看负责班级的学生、考试和成绩 |
| `demo-math` | 任课教师，查看任教科目和相关阅卷任务 |
| `demo-grade-leader` | 年级组长，按年级作用域查看数据 |
| `demo-subject-leader` | 学科组长，按科目作用域查看数据 |
| `demo-ai` | AI 阅卷教师，默认匿名且只读 |

## 使用建议

1. 用默认管理员创建和配置考试。
2. 在考试概览中配置试题编写人、扫描人和阅卷人。
3. 切换到试题编写人，验证是否只能进入设计与印刷流程。
4. 切换到扫描人，验证是否只能进入扫描绑定流程。
5. 切换到阅卷人或 AI 教师，验证匿名阅卷界面。

## 注意事项

- 如果身份选择器保存了非法值，前端会清理该值并回到默认管理员。
- 成绩发布只影响成绩可见性，不自动授予学生身份字段可见性。
- 阅卷人和 AI 教师默认不接收 `student_id`、姓名、学号、班级、整卷原图或答题卡绑定信息。
