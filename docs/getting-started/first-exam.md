# 完成第一场考试

本教程用一套固定示例，带你从 0 完成一场考试：准备基础数据、新建考试、编写试题、生成答题卡、上传扫描件、完成阅卷并发布成绩。

!!! note "示例数据说明"
    本页中的学生、题目和分数只用于教学演示，不代表任何真实学校数据。

## 适用角色

- 学校管理员或教务人员。
- 考试创建人或考官。
- 试题编写人。
- 扫描人员。
- 阅卷教师。

如果你一个人正在试用系统，可以使用管理员账号依次完成全部步骤。

## 你将完成什么

- 在 `demoSchool` 组织中准备一套最小基础数据。
- 创建 `2026 春季 七年级数学期中考试`。
- 配置一张 30 分试卷。
- 生成试卷和答题卡。
- 上传扫描件并处理识别结果。
- 完成客观复核和主观阅卷。
- 发布成绩并查看报告入口。

## 开始前需要

- 组织入口可用：`/org/demoSchool/grading`。
- 你已经登录，并具备管理员或考试创建权限。
- 学校至少允许你维护基础数据和创建考试。

## 1. 准备基础数据

进入 **管理员设置**，按顺序检查或创建基础数据。

建议顺序：

1. 在学校基础信息中确认学校名称。
2. 在校历设置中创建 `2026 学年` 和 `春季学期`。
3. 在年级设置中创建 `七年级`。
4. 在科目设置中创建 `数学`。
5. 在班级管理中创建 `七年级 1 班`。
6. 在学生管理中添加 3 名学生：张一、李二、王三。
7. 在教师通讯录或教师管理中确认至少有一名可作为考试创建人、试题编写人、扫描人和阅卷人的教师。

!!! note "图 2-1 管理员设置入口（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-01-admin-settings-entry.png`

    画面要求：显示侧边栏中的“管理员设置”，并能看到学校基础信息、校历、年级或科目相关入口。

完成后你应该看到：

- 校历中存在 `2026 学年 / 春季学期`。
- 年级中存在 `七年级`。
- 科目中存在 `数学`。
- `七年级 1 班` 中有张一、李二、王三。

常见问题：

- **找不到管理员设置**：当前账号没有对应管理权限，请联系学校管理员。
- **无法新建科目或年级**：检查是否拥有科目或年级管理权限。
- **学生没有出现在班级中**：确认学生状态为可用，并且已经绑定到 `七年级 1 班`。

## 2. 新建考试

进入 **考试管理**，创建考试。

填写建议：

| 字段 | 填写 |
| --- | --- |
| 学年 | `2026 学年` |
| 学期 | `春季学期` |
| 年级 | `七年级` |
| 科目 | `数学` |
| 考试标题 | `2026 春季 七年级数学期中考试` |

!!! note "图 2-2 新建考试入口（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-02-create-exam-entry.png`

    画面要求：显示考试管理页面中新建考试的位置。具体按钮名称以后续界面为准。

完成后你应该看到：

- 考试出现在考试管理列表中。
- 进入考试后可以看到考试工作区。
- 考试工作区至少包含概览页。

常见问题：

- **学年、学期、年级或科目下拉为空**：先回到管理员设置补齐基础数据。
- **保存失败**：检查标题是否为空，并确认当前账号有创建考试权限。

## 3. 配置考试班级和协作人员

进入考试工作区的 **概览**。

建议配置：

- 考核班级：`七年级 1 班`。
- 试题编写人：负责维护题目和答题卡。
- 扫描人：负责上传扫描件和处理识别异常。
- 阅卷人：负责客观复核和主观阅卷。

如果你是管理员试用系统，可以先把这些角色配置给同一个测试教师。

!!! note "图 2-3 考试协作区域（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-03-exam-collaboration.png`

    画面要求：显示考试概览中的协作人员和考核班级区域。

完成后你应该看到：

- 考核班级中包含 `七年级 1 班`。
- 协作人员列表中有试题编写人、扫描人和阅卷人。
- 对应教师登录后能看到自己的工作入口。

常见问题：

- **某位教师不在候选列表中**：先确认教师档案状态可用。
- **阅卷教师看不到任务**：只配置阅卷人还不够，后续还需要扫描交付和阅卷分配。

## 4. 编写试题

进入 **考试工作区 > 设计与印刷 > 试题配置**。

创建一份 30 分试卷：

| 题型 | 数量 | 分值 | 小计 |
| --- | --- | --- | --- |
| 选择题 | 5 | 每题 2 分 | 10 分 |
| 填空题 | 2 | 每题 5 分 | 10 分 |
| 主观题 | 1 | 10 分 | 10 分 |

建议操作：

1. 设置试卷标题为 `2026 春季 七年级数学期中考试`。
2. 新增选择题题组，创建 5 道选择题。
3. 为每道选择题填写选项和正确答案。
4. 新增填空题题组，创建 2 道填空题。
5. 新增主观题题组，创建 1 道 10 分主观题。
6. 填写评分标准。
7. 保存试题。
8. 使用在线预览检查试卷和答题卡版式。

!!! note "图 2-4 试题配置页面（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-04-paper-editor.png`

    画面要求：显示题组、题目编辑器、分值和预览入口的位置。

完成后你应该看到：

- 试卷总分为 30 分。
- 题号连续，没有重复或缺失。
- 答题卡中有选择题填涂区、填空题答题区和主观题答题区。

常见问题：

- **总分不等于 30**：检查每道题分值。
- **答题卡区域不符合预期**：调整题型、答题区域或题目顺序后重新预览。
- **进入预印前发现题目错误**：继续编辑并保存；进入预印后再修改可能需要重新生成印刷数据。

## 5. 预印和生成印刷数据

进入 **考试工作区 > 设计与印刷 > 预印**。

检查：

- 试卷纸张。
- 答题卡纸张。
- 单面或双面。
- 空白页策略。
- 答题卡增强扫描设置。

预览无误后，进入 **印刷数据**，生成试卷 PDF 和答题卡整包。

!!! note "图 2-5 预印配置（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-05-preprint-settings.png`

    画面要求：显示预印配置中的纸张、单双面、空白页策略和预览通过位置。

完成后你应该看到：

- 当前考试有可下载的试卷 PDF。
- 当前考试有可下载的答题卡 PDF。
- 答题卡带有条码和页码。

常见问题：

- **无法生成印刷数据**：确认预印已经通过。
- **打印前又修改了题目**：需要重新确认预印和印刷数据，避免旧答题卡与新试卷不匹配。

## 6. 打印和组织考试

下载 PDF 后按学校流程打印。

建议：

- 使用系统生成的答题卡，不要手工重画答题区域。
- 打印后检查条码、页码、学号填涂区和题目答题区是否清晰。
- 考试结束后按班级或考场整理答题卡。

完成后你应该得到：

- 学生已作答的纸质答题卡。
- 条码和页码完整的扫描材料。

常见问题：

- **条码打印太淡**：重新打印答题卡，避免后续识别失败。
- **答题卡与当前考试不一致**：不要上传到当前考试，先确认是否来自其他考试或旧印刷版本。

## 7. 上传扫描件

考试结束后，进入 **考试工作区 > 答题卡扫描 > 扫描批次**。

操作：

1. 准备扫描 PDF、JPG 或 PNG。
2. 上传扫描件。
3. 等待系统创建扫描批次并处理页面。
4. 查看录入结果、答题卡统计和异常提示。
5. 对学号识别失败或重复绑定的答题卡进行人工处理。

!!! note "图 2-6 扫描批次页面（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-06-scan-batches.png`

    画面要求：显示扫描批次、上传入口和录入结果入口。

完成后你应该看到：

- 扫描批次已经生成。
- 答题卡按条码和页码归组。
- 学生绑定成功，或异常卷进入人工处理。
- 客观题识别结果和主观题裁剪区域已经生成。

常见问题：

- **条码识别失败**：检查条码是否模糊、缺失或被遮挡。
- **学号绑定失败**：进入录入结果，人工选择正确学生。
- **上传错考试**：不要继续交付阅卷；先删除或隔离错误批次，并重新上传正确答题卡。

更多处理方法见 [扫描与识别 SOP](../scanning-printing/sop.md)。

## 8. 交付阅卷和分配任务

在扫描结果确认后，进入 **考试工作区 > 答题卡扫描 > 交付阅卷**。

建议操作：

1. 检查异常卷是否已经处理。
2. 确认可交付的答题卡范围。
3. 交付阅卷。
4. 进入 **阅卷 > 阅卷分配**，给阅卷教师分配题目。

完成后你应该看到：

- 主观题和填空题生成阅卷任务。
- 阅卷教师能进入自己的任务。
- 未交付或未分配时，阅卷教师不会看到可批改内容。

常见问题：

- **主观阅卷没有任务**：确认扫描已经交付阅卷，并且题目已经分配给阅卷教师。
- **某位教师看不到某道题**：检查该题是否分配给该教师。

## 9. 完成客观复核和主观阅卷

进入 **考试工作区 > 阅卷**。

操作：

- 在 **客观复核** 中处理低置信度、空白、多选或识别失败的客观题。
- 在 **主观阅卷** 中录入填空题和主观题分数。
- 按评分标准录入评语或说明。
- 保存后检查任务状态。

!!! note "图 2-7 客观复核和主观阅卷（截图待上传）"
    建议文件：`docs/assets/screenshots/02-first-exam/02-07-grading-review.png`

    画面要求：显示客观复核、主观阅卷和保存评分区域。

完成后你应该看到：

- 客观题异常已经复核。
- 填空题和主观题已有分数。
- 阅卷任务状态不再为空。

常见问题：

- **看不到学生姓名**：这是匿名阅卷设计。普通阅卷教师默认只看到匿名答案和题目裁剪区域。
- **分数无法保存**：检查是否被分配到这道题，或是否已超过允许分值。

## 10. 发布成绩和查看报告

进入 **考试工作区 > 成绩与报告 > 成绩**。

操作：

1. 检查每名学生总分。
2. 必要时重新计算成绩。
3. 确认无误后发布成绩。
4. 进入 **报告** 查看可用报告入口。
5. 按权限导出 CSV 或 Excel。

!!! info "当前版本状态"
    当前版本支持成绩汇总、成绩发布和基础报告查看。更完整的家长端报告、批量报告推送和自动化分析以后续版本为准。

完成后你应该看到：

- 成绩状态变为已发布。
- 具备成绩查看权限的教师可以查看成绩。
- 导出字段会根据当前账号权限裁剪。

常见问题：

- **发布后仍看不到学生姓名**：成绩发布不等于开放学生身份字段。
- **导出没有姓名或学号**：当前账号没有对应学生身份字段权限。
- **总分不正确**：检查客观复核、主观阅卷和调整分后重新计算。

## 完整检查清单

完成第一场考试后，建议检查：

- [ ] 基础数据完整：学年、学期、年级、班级、学生、教师、科目。
- [ ] 考试已经创建，并包含 `七年级 1 班`。
- [ ] 协作人员已经配置。
- [ ] 试卷总分为 30 分。
- [ ] 试卷 PDF 和答题卡 PDF 已生成。
- [ ] 扫描批次已处理，异常卷已确认。
- [ ] 阅卷任务已生成并完成。
- [ ] 成绩已发布。

## 每一步检查清单

| 步骤 | 检查项 |
| --- | --- |
| 准备基础数据 | 学年、学期、年级、班级、学生、教师、科目都存在且状态可用 |
| 新建考试 | 考试标题、学期、年级、科目正确，进入考试工作区后信息一致 |
| 配置协作 | 考核班级和协作人员已保存，相关教师能看到对应入口 |
| 编写试题 | 题号连续，总分 30 分，客观题答案、填空题答案和主观题评分标准完整 |
| 预印印刷 | 预览通过，试卷 PDF 和答题卡 PDF 来自当前 active 版本 |
| 打印考试 | 条码、页码、学号填涂区和答题区域清晰，无旧版本答题卡混入 |
| 上传扫描件 | 扫描批次完成，条码归组和学生绑定正常，异常卷已处理 |
| 交付阅卷 | 扫描结果已交付，主观题和填空题已经分配给阅卷教师 |
| 完成阅卷 | 客观异常已复核，主观题已保存分数，任务状态正常 |
| 发布成绩 | 成绩已重新计算，发布前无缺失分数，导出字段符合权限预期 |

## 正常结果和异常结果对照

| 环节 | 正常结果 | 异常结果 | 处理方向 |
| --- | --- | --- | --- |
| 基础数据 | 新建考试可选择学期、年级、科目 | 下拉为空 | 回到管理员设置补数据 |
| 协作配置 | 教师看到对应模块 | 教师看不到入口 | 检查是否加入考试协作或是否有权限 |
| 试题配置 | 试卷总分和答题卡区域正确 | 总分不对或区域不够 | 回到试题配置调整分值和答题样式 |
| 印刷数据 | PDF 可下载且条码清晰 | 无法生成或条码不清晰 | 检查预印状态并重新生成或打印 |
| 扫描上传 | 扫描批次完成，学生绑定正常 | 条码失败、学号失败、缺页 | 按扫描 SOP 处理异常 |
| 阅卷 | 阅卷教师看到分配任务 | 任务为空 | 检查交付阅卷和阅卷分配 |
| 成绩发布 | 成绩状态为已发布 | 无法发布或总分异常 | 检查阅卷完成度、重新计算成绩 |
