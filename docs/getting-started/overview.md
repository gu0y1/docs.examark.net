# Examark 是什么

Examark 是面向学校纸笔考试流程的考试管理与智能阅卷系统。它帮助学校把创建考试、配置试题、生成答题卡、扫描上传、识别复核、主观阅卷、成绩发布和报告导出串成一条可追踪的流程。

## 适用角色

- 学校管理员和教务人员。
- 考试创建人、考官和年级/学科负责人。
- 试题编写人。
- 扫描人员。
- 阅卷教师。
- IT / 运维人员。

## 你将完成什么

使用 Examark 组织一场考试时，通常会完成这些事情：

1. 管理员先维护学年、学期、年级、班级、学生、教师和科目。
2. 考试创建人新建考试，选择考试所属学期、年级和科目。
3. 考试创建人配置考试班级和协作人员。
4. 试题编写人维护题目、分值、答案和评分标准。
5. 试题编写人或考试创建人生成试卷 PDF 和答题卡 PDF。
6. 扫描人员上传考试后的扫描件。
7. 系统识别条码、学号填涂、客观题填涂，并裁剪填空题和主观题区域。
8. 阅卷教师完成客观复核和主观阅卷。
9. 考试创建人确认成绩并发布。

## 开始前需要

- 学校已经有可用的组织入口，例如 `/org/demoSchool/grading`。
- 当前账号已经登录，并具备对应任务的权限。
- 学校基础数据至少包含学年、学期、年级、班级、学生、教师和科目。

如果你还没有任何数据，先从 [完成第一场考试](first-exam.md) 的示例教程开始。

## 三类关键数据

### 学校基础数据

学校基础数据决定系统里有哪些人、班级和科目。管理员通常先维护：

- 学年和学期。
- 年级和班级。
- 学生与家长档案。
- 教师、角色和教师绑定。
- 科目。

### 考试内协作

一位教师在学校里可能是普通任课教师，但在某场考试里可以被指定为试题编写人、扫描人或阅卷人。考试内角色只在当前考试中生效。

常见考试内角色包括：

- 创建人。
- 考官。
- 试题编写人。
- 扫描人。
- 阅卷人。

### 考试处理流程

考试处理从“新建考试”开始，到“成绩发布与报告”结束。系统会把试卷、答题卡、扫描页、识别结果、阅卷任务和成绩记录关联起来，便于后续追踪和审计。

## 完成后你应该知道

读完本页后，你应该能判断：

- 自己属于哪个角色。
- 开始一场考试前需要准备哪些基础数据。
- 哪些操作属于全校管理，哪些操作只属于单场考试。
- 为什么同一个老师在不同考试里看到的入口可能不同。

## 常见问题

**我能不能直接创建考试，不先维护基础数据？**

不建议。新建考试至少需要学年、学期、年级和科目；参加考试的学生需要来自班级和学生档案。

**我能看到某个页面，就一定能保存吗？**

不一定。前端会尽量只展示可用入口，但后端仍会按权限再次校验。没有权限时，保存、导出或查看敏感字段会被拒绝。
