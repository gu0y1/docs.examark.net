# 系统概览

Examark 将学校考试过程拆为三层：学校主数据、考试内协作、考试处理流程。

## 学校主数据

学校主数据决定系统中有哪些基础对象：

- 学年和学期。
- 年级、班级和学生。
- 科目。
- 教师、AI 教师和教师绑定。
- 角色和权限。
- 学生家长档案。

这些数据通常由管理员或教务人员维护。考试创建、班级范围、教师作用域和成绩可见范围都会依赖这些基础数据。

## 考试内协作

每一场考试都有独立的协作人员：

- 创建人 owner。
- 考官 examiner。
- 试题编写人 paper_author。
- 扫描人 scanner。
- 阅卷人 grader。
- 考核班级 ExamClassroom。
- 题目分配 ExamQuestionAssignment。

考试内角色只在当前考试生效，不等同于学校级角色。

## 考试处理流程

考试处理从“新建考试”开始，到“成绩发布与报告”结束。系统会把试卷、答题卡、扫描页、识别结果、阅卷任务和成绩记录关联起来，便于后续追踪和审计。
