# 常见问题

## 为什么我看不到某个考试入口？

考试工作区入口按当前账号 capability 裁剪。比如扫描人默认看不到设计与印刷，阅卷人默认看不到扫描原卷和成绩报告。

## 配置考试协作时显示 Not Found 怎么办？

请确认前端和后端都已更新到最新代码，并优先通过同源前端地址访问。默认管理员会在候选人接口不可用时回退到教师列表。

## 为什么阅卷人看不到学生姓名？

这是匿名阅卷设计。普通阅卷人和 AI 教师默认只能看到匿名答案、题目裁剪图、识别文本和评分标准。

## 成绩发布后为什么仍看不到学生身份？

成绩发布只开放成绩查看，不自动开放学生身份字段。身份字段由单独 capability 控制。

## scanner 能看到什么？

scanner 默认可在扫描绑定流程看到姓名、学号、班级、扫描原卷和答题卡绑定关系，但不能看敏感字段和成绩报告。

## WSL 1 报错怎么办？

Next.js 当前环境可能要求 WSL 2 或更高版本。升级 WSL 后再运行前端构建或开发服务器。

## 3000 端口不可用怎么办？

Next dev server 会尝试使用可用端口，例如 3001。浏览器需要打开实际日志中显示的 Local 地址。

## 扫描失败怎么办？

先确认答题卡是否由当前系统生成，并检查：

- 条码是否清晰。
- 页码是否完整。
- 扫描件是否过度倾斜或模糊。
- 是否上传了正确考试的答题卡。

## 导出为什么没有姓名和学号？

导出同样遵守后端字段裁剪。如果当前账号没有学生身份 capability，导出只包含匿名 ID 和成绩字段。
