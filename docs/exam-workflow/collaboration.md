# 配置考试协作

## 适用角色

- 创建人 owner。
- 考官 examiner。
- 超级管理员。

## 你将完成什么

- 指定本场考试的参与人。
- 明确谁负责试题、扫描、阅卷和成绩流程。
- 避免把学校级角色和单场考试角色混在一起。

## 开始前需要

- 考试已经创建。
- 参与教师已经存在于教师通讯录中。
- 当前账号是本场考试创建人、考官或系统管理员。

## 入口

**考试工作区 > 概览 > 考试协作**

## 考试内角色

| 角色 | 能做什么 | 默认不能做什么 |
| --- | --- | --- |
| 创建人 owner | 管理本场考试全部流程 | 不适用 |
| 考官 examiner | 管理大部分流程、参与人、题目分配和成绩发布 | 不转移 owner，不删除考试 |
| 试题编写人 paper_author | 试题设计、预印、印刷数据 | 不看身份、扫描、绑定、成绩 |
| 扫描人 scanner | 扫描上传、扫描原卷、答题卡绑定 | 不编辑试题、不阅卷、不发布成绩 |
| 阅卷人 grader | 客观复核、主观阅卷 | 默认匿名，不看学生身份 |

## 操作步骤

1. 进入考试工作区的概览页。
2. 找到考试协作区域。
3. 选择试题编写人、扫描人和阅卷人。
4. 保存配置。
5. 让对应教师重新进入考试工作区，确认入口是否出现。

## 配置建议

- 小型测试可以先由同一个管理员承担多个角色。
- 正式考试建议试题编写、扫描和阅卷分工明确。
- 阅卷人还需要在阅卷分配中分配题目，才会看到具体任务。

## 放权原则

创建人或考官可以对指定参与人额外授予身份信息能力。敏感字段查看能力只应授予确实需要处理学生身份的人员。

放权会写入审计日志。审计记录字段名、资源和动作，不记录身份证号、电话、地址、家长信息等敏感明文。

## 完成后你应该看到

- 考试概览中能看到本场考试的协作人员。
- 试题编写人能进入设计与印刷。
- 扫描人能进入答题卡扫描。
- 阅卷人会在任务交付和分配后看到阅卷入口。

## 常见问题

**配置了阅卷人，为什么对方还没有任务？**

阅卷任务需要在扫描交付和阅卷分配后生成。只配置考试协作不会自动生成题目任务。

**扫描人能不能看到成绩？**

默认不能。扫描人需要学生绑定信息和扫描原卷，但不默认拥有成绩查看权限。

**试题编写人能不能看学生姓名？**

默认不能。试题编写只需要试题、试卷和答题卡，不需要学生身份信息。
