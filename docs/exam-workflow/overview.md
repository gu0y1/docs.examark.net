# 考试流程总览

Examark 的考试流程围绕一场考试展开。每个环节都有明确的参与人、数据产物和权限规则。

## 适用角色

- 考试创建人。
- 考官。
- 教务人员。
- 第一次参与 Examark 考试流程的教师。

## 你将完成什么

- 了解一场考试从创建到发布成绩的完整顺序。
- 判断每个环节应该由谁负责。
- 知道每个阶段完成后应该产生什么数据。

## 流程表

| 阶段 | 主要参与人 | 产物 |
| --- | --- | --- |
| 准备基础数据 | 管理员、教务人员 | 学年、学期、年级、班级、学生、教师、科目 |
| 新建考试 | 管理员、教务长、创建人 | 考试记录、学期、科目、年级快照 |
| 配置协作 | 创建人、考官、超管 | 考试参与人、考核班级、题目分配 |
| 试题编写 | 创建人、考官、试题编写人 | 试卷、题组、题目、评分标准 |
| 预印 | 创建人、考官、试题编写人 | 预印版本、纸张和单双面配置 |
| 印刷数据 | 创建人、考官、试题编写人 | 试卷 PDF、答题卡 PDF、条码 |
| 扫描上传 | 扫描人、创建人、考官 | 扫描批次、扫描页、识别结果 |
| 阅卷 | 阅卷人、创建人、考官 | 客观复核结果、主观评分、评语 |
| 成绩发布 | 创建人、考官、超管 | 成绩记录、导出、报告入口 |

## 推荐操作顺序

1. 管理员先准备学年、学期、年级、班级、学生、教师和科目。
2. 考试创建人新建考试。
3. 考试创建人配置考核班级和协作人员。
4. 试题编写人完成试题配置。
5. 试题编写人或考试创建人完成预印和印刷数据。
6. 扫描人员上传考试后的答题卡扫描件。
7. 扫描人员处理识别异常和人工绑定。
8. 考试创建人或考官交付阅卷并分配题目。
9. 阅卷教师完成客观复核和主观阅卷。
10. 考试创建人或考官确认成绩并发布。

## 状态流转

试卷进入预印后会锁定当前编辑版本。需要修改时，应先废弃当前预印或印刷版本，再回到试题配置继续编辑。

已有扫描批次后，不应随意更换印刷版本。扫描识别依赖当前 active 印刷版本和答题卡条码。

## 权限原则

- 能编辑试卷不代表能查看学生身份。
- 能扫描不代表能阅卷。
- 能阅卷不代表能查看学生姓名、学号或整卷原图。
- 成绩发布只改变成绩查看状态，不自动开放身份字段。

## 完成后你应该知道

- 一场考试最少需要基础数据、考试记录、试题、印刷数据、扫描件、阅卷结果和成绩记录。
- 扫描完成不代表阅卷任务已经分配。
- 成绩发布不代表学生身份字段自动开放。

## 常见问题

**可以跳过预印直接扫描吗？**

不建议。扫描识别依赖当前 active 印刷版本和答题卡条码。跳过或混用版本会导致条码、页码或题目区域不匹配。

**为什么扫描完成后阅卷人仍没有任务？**

扫描完成只是生成识别结果。还需要交付阅卷，并在阅卷分配中把题目分配给阅卷教师。

**为什么发布成绩后仍有人看不到姓名？**

成绩查看和学生身份字段是两套权限。发布成绩不会自动开放姓名、学号、班级或扫描原卷。
