# 考试流程总览

Examark 的考试流程围绕一场考试展开。每个环节都有明确的参与人、数据产物和权限规则。

## 流程表

| 阶段 | 主要参与人 | 产物 |
| --- | --- | --- |
| 新建考试 | 管理员、教务长、创建人 | 考试记录、学期、科目、年级快照 |
| 配置协作 | 创建人、考官、超管 | 考试参与人、考核班级、题目分配 |
| 试题编写 | 创建人、考官、试题编写人 | 试卷、题组、题目、评分标准 |
| 预印 | 创建人、考官、试题编写人 | 预印版本、纸张和单双面配置 |
| 印刷数据 | 创建人、考官、试题编写人 | 试卷 PDF、答题卡 PDF、条码 |
| 扫描上传 | 扫描人、创建人、考官 | 扫描批次、扫描页、识别结果 |
| 阅卷 | 阅卷人、创建人、考官 | 客观复核结果、主观评分、评语 |
| 成绩发布 | 创建人、考官、超管 | 成绩记录、导出、学生报告 |

## 状态流转

试卷进入预印后会锁定当前编辑版本。需要修改时，应先废弃当前预印或印刷版本，再回到试题配置继续编辑。

## 权限原则

- 能编辑试卷不代表能查看学生身份。
- 能扫描不代表能阅卷。
- 能阅卷不代表能查看学生姓名、学号或整卷原图。
- 成绩发布只改变成绩查看状态，不自动开放身份字段。
