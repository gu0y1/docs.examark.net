# 权限模型概览

Examark 的权限模型用于保护学生身份、扫描原卷和成绩数据。它不是只看“是不是管理员”，而是分层判断当前账号能做什么、能看什么。

## 适用角色

- 学校管理员。
- 考试创建人。
- 教务人员。
- 对权限显示有疑问的教师。

## 你将完成什么

- 理解学校级角色、考试内角色和身份字段权限的区别。
- 判断为什么自己能进入某个流程，但看不到某些学生信息。

## 三层权限

Examark 将权限分为三层：

1. 学校级角色权限。
2. 考试内流程角色。
3. 身份信息 capability。

这样可以避免“学校里是什么角色”和“这场考试里能做什么”混在一起。

## 学校级角色

学校级角色控制全校数据视野和主数据维护能力。例如管理员、教务长、班主任、任课教师、年级组长、学科组长。

## 考试内角色

考试内角色只在单场考试中生效。例如某位教师可能是学校的普通任课教师，但在某场考试中被指定为试题编写人或阅卷人。

## 身份信息 capability

身份信息 capability 决定是否能看到学生身份、扫描原卷、绑定关系和敏感字段。

核心原则：

- 能阅卷不代表能看身份。
- 能看成绩不代表能看扫描原卷。
- 能扫描不代表能在阅卷界面去匿名。

## 完成后你应该知道

- 学校里的职务不等于每场考试中的角色。
- 一场考试里的流程权限不等于学生身份字段权限。
- 前端隐藏入口只是体验优化，后端仍会做最终权限校验。

## 常见问题

**我为什么可以上传扫描件，却不能发布成绩？**

扫描和成绩发布是不同流程能力。扫描人员默认只处理扫描和绑定，不负责成绩发布。

**我为什么有管理员设置入口，但只看到部分标签？**

管理员设置按具体管理权限显示。只有校历权限时只看到校历设置；没有角色管理权限时不会看到角色管理。
