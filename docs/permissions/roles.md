# 学校级角色与考试内角色

## 学校级角色

| 角色 | 典型能力 |
| --- | --- |
| 管理员 | 全校管理、角色权限、审计和全部考试流程 |
| 教务长 | 维护教学主数据、考试、印刷和成绩流程 |
| 班主任 | 查看负责班级学生、家长、考试和成绩 |
| 任课教师 | 查看任教班级和科目的考试、成绩、阅卷任务 |
| 年级组长 | 按年级作用域查看考试和成绩 |
| 学科组长 | 按科目作用域查看考试、阅卷和成绩 |
| AI 教师 | 预留 AI 阅卷和辅助分析能力 |

## 角色对照表

下表用于学校上线时讨论“谁应该看到什么”。实际权限仍以学校配置为准。

| 学校身份 | 常见作用域 | 通常负责 | 默认不应开放 |
| --- | --- | --- | --- |
| 学校管理员 | 全校 | 初始化数据、配置角色、处理高风险权限 | 不适用 |
| 教务长 | 全校或年级 | 校历、年级、科目、考试流程、成绩发布 | 不必要的敏感字段明文 |
| 年级组长 | 年级 | 本年级考试进度、成绩汇总、教师协调 | 其他年级学生身份 |
| 学科组长 | 科目 | 本科目试题、阅卷、成绩分析 | 其他科目考试管理 |
| 班主任 | 班级 | 本班学生、成绩和家校沟通 | 非本班敏感信息 |
| 任课教师 | 班级 + 科目 | 任教班级考试、阅卷、成绩查看 | 与任教无关的班级身份字段 |
| 扫描人员 | 考试内 | 扫描上传、学生绑定、异常卷处理 | 成绩报告和敏感学生字段 |
| 阅卷教师 | 考试内题目 | 客观复核、主观评分 | 学生姓名、学号、整卷原图 |

## 考试内角色

| 角色 | 流程能力 |
| --- | --- |
| owner | 本场考试最高权限 |
| examiner | 管理大部分考试流程、参与人、题目分配、成绩发布 |
| paper_author | 编辑试题、预印、生成印刷数据 |
| scanner | 扫描上传、扫描原卷和答题卡绑定流程 |
| grader | 客观复核和主观阅卷 |

## 权限计算

后端统一通过 `get_exam_capabilities(actor, exam)` 计算能力。前端只根据该结果展示入口，后端每个写接口仍会强校验。

## 使用建议

- 学校级角色用于长期职责，例如班主任、年级组长和教务长。
- 考试内角色用于单场考试协作，例如某场考试的扫描人或阅卷人。
- 学生身份字段权限要单独讨论，不要因为某人能阅卷就自动开放姓名和学号。
- 高风险能力应只给少数确实需要处理争议、审计或管理员配置的人。
