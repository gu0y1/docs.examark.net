# 学校基础数据

学校基础数据决定 Examark 中有哪些学年、班级、学生、教师和科目。考试创建、班级范围、教师作用域和成绩可见范围都会依赖这些数据。

## 适用角色

- 学校管理员。
- 教务人员。
- 具备对应管理员设置权限的教师。

## 你将完成什么

- 初始化学校基础信息。
- 维护校历、年级、科目和班级。
- 为后续考试创建准备基础数据。

## 开始前需要

- 当前账号能看到 **管理员设置**。
- 如果只能看到部分标签，说明当前账号只拥有部分管理权限。

## 推荐维护顺序

1. 在学校基础信息中确认学校名称和简称。
2. 创建学年。
3. 在学年下创建学期。
4. 创建年级。
5. 创建科目。
6. 创建班级并绑定年级。
7. 维护学生和教师数据。

## 学校基础信息

学校名称和简称会出现在页面顶栏、部分打印件和报告场景中。

如果学校启用了页面水印，业务页面会叠加组织与账号水印，便于追责。

## 校历

校历包含学年和学期。新建考试时必须选择已有学期。

建议：

- 学年名称使用学校内部一致的命名方式。
- 学期排序保持清晰。
- 已被考试引用的学期不要随意删除。

## 年级

年级用于班级和考试。年级可参与轮转，例如七年级升到八年级。

维护年级时，注意历史考试会保留创建时的年级语义，不会因为后续升年级自动改变历史考试名称。

## 科目

科目用于考试创建、教师绑定和成绩归属。

被考试引用的科目不应直接删除，通常转为停用或保留历史记录。

## 班级

班级属于某个年级，是学生管理和考试考核班级的基础。

班级变更会影响后续考试选择班级，但历史考试应保留当时的考试范围。

## 完成后你应该看到

- 新建考试时可以选择正确的学年、学期、年级和科目。
- 班级管理中能看到对应年级下的班级。
- 学生可以被添加到正确班级。

## 常见问题

**为什么新建考试时没有学期可选？**

先在校历设置中创建学年和学期。

**为什么教师看不到管理员设置？**

管理员设置按权限显示。当前教师需要至少拥有一个设置相关权限，例如校历、年级、科目、角色、教师或学生管理权限。

**可以删除已经被考试使用的科目吗？**

不建议。删除被引用的数据会影响历史追踪。通常应停用而不是删除。
