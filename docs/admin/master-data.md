# 学校基础数据

学校基础数据决定 Examark 中有哪些学年、班级、学生、教师和科目。考试创建、班级范围、教师作用域和成绩可见范围都会依赖这些数据。

## 适用角色

- 学校管理员。
- 教务人员。
- 具备对应管理员设置权限的教师。

## 你将完成什么

- 初始化学校基础信息。
- 维护校历、年级、科目和班级。
- 为后续考试创建准备基础数据。

## 开始前需要

- 当前账号能看到 **管理员设置**。
- 如果只能看到部分标签，说明当前账号只拥有部分管理权限。

## 学校上线 SOP

首次上线一个学校时，建议管理员按下面顺序完成初始化。不要先创建考试再回头补基础数据，否则新建考试、协作配置和成绩归属都容易缺少可选项。

| 顺序 | 任务 | 操作位置 | 完成标准 |
| --- | --- | --- | --- |
| 1 | 初始化学校基础信息 | 管理员设置 > 学校基础信息 | 学校名称、简称和基础显示信息正确 |
| 2 | 创建学年与学期 | 管理员设置 > 校历设置 | 至少有当前学年和当前学期 |
| 3 | 创建年级 | 管理员设置 > 年级设置 | 当前教学年级都已存在 |
| 4 | 创建科目 | 管理员设置 > 科目设置 | 考试会用到的科目都已存在 |
| 5 | 创建班级 | 班级管理 | 班级已绑定正确年级 |
| 6 | 创建学生 | 学生管理 | 学生已进入正确班级 |
| 7 | 创建教师 | 教师通讯录或管理员设置 > 教师管理 | 教师姓名、工号、状态正确 |
| 8 | 配置班主任 | 教师管理 | 班主任绑定到对应班级 |
| 9 | 配置任课教师 | 教师管理 | 任课教师绑定到班级和科目 |
| 10 | 配置年级组长和学科组长 | 教师管理 | 作用域绑定到对应年级或科目 |
| 11 | 配置角色与权限 | 管理员设置 > 角色管理 | 教务、教师、扫描、阅卷等角色可用 |
| 12 | 创建首场测试考试 | 考试管理 | 能从考试创建跑到成绩发布 |

## 上线前检查清单

- [ ] 学校名称和简称正确。
- [ ] 当前学年和学期已创建。
- [ ] 当前年级、班级和科目已创建。
- [ ] 每个班级至少有可用学生。
- [ ] 每位教师都有唯一工号。
- [ ] 班主任、任课教师、年级组长、学科组长的作用域绑定正确。
- [ ] 教务人员能看到需要的管理员设置标签。
- [ ] 普通教师不会看到不该看到的敏感字段。
- [ ] 使用测试考试验证创建、扫描、阅卷和成绩发布流程。

## 推荐维护顺序

1. 在学校基础信息中确认学校名称和简称。
2. 创建学年。
3. 在学年下创建学期。
4. 创建年级。
5. 创建科目。
6. 创建班级并绑定年级。
7. 维护学生和教师数据。

## 学校基础信息

学校名称和简称会出现在页面顶栏、部分打印件和报告场景中。

如果学校启用了页面水印，业务页面会叠加组织与账号水印，便于追责。

## 校历

校历包含学年和学期。新建考试时必须选择已有学期。

建议：

- 学年名称使用学校内部一致的命名方式。
- 学期排序保持清晰。
- 已被考试引用的学期不要随意删除。

## 年级

年级用于班级和考试。年级可参与轮转，例如七年级升到八年级。

维护年级时，注意历史考试会保留创建时的年级语义，不会因为后续升年级自动改变历史考试名称。

## 科目

科目用于考试创建、教师绑定和成绩归属。

被考试引用的科目不应直接删除，通常转为停用或保留历史记录。

## 班级

班级属于某个年级，是学生管理和考试考核班级的基础。

班级变更会影响后续考试选择班级，但历史考试应保留当时的考试范围。

## 角色与权限准备

学校上线前，建议至少准备这些角色或等价权限组合：

| 角色 | 建议用途 |
| --- | --- |
| 学校管理员 | 管理全校配置、角色、教师、学生和审计 |
| 教务长 | 维护校历、年级、科目、考试和成绩流程 |
| 年级组长 | 查看和管理本年级范围内的考试和成绩 |
| 学科组长 | 查看和管理本科目相关考试和阅卷 |
| 班主任 | 查看本班学生、考试和成绩 |
| 任课教师 | 参与任教班级和科目的考试、阅卷和成绩查看 |
| 扫描人员 | 处理答题卡扫描、学生绑定和扫描原卷 |

实际名称可以按学校制度调整，但不要把所有教师都配置成系统管理员。

## 完成后你应该看到

- 新建考试时可以选择正确的学年、学期、年级和科目。
- 班级管理中能看到对应年级下的班级。
- 学生可以被添加到正确班级。

## 常见问题

**为什么新建考试时没有学期可选？**

先在校历设置中创建学年和学期。

**为什么教师看不到管理员设置？**

管理员设置按权限显示。当前教师需要至少拥有一个设置相关权限，例如校历、年级、科目、角色、教师或学生管理权限。

**可以删除已经被考试使用的科目吗？**

不建议。删除被引用的数据会影响历史追踪。通常应停用而不是删除。
