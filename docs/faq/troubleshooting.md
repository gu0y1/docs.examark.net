# 故障处理

本页用于处理日常考试流程中常见的失败情况。技术部署、服务器和开发环境问题请查看 [Linux 部署与排错](../technical/deployment.md)。

## 适用角色

- 考试创建人。
- 扫描人员。
- 阅卷教师。
- 教务人员。

## 找不到考试

可能原因：

- 选错学年、学期或科目筛选。
- 当前账号没有查看该考试的权限。
- 你没有被加入本场考试协作。

处理步骤：

1. 回到考试管理。
2. 清空或调整筛选条件。
3. 确认考试是否存在于正确学期和科目下。
4. 联系考试创建人确认你是否被加入考试。

## 没有权限

可能原因：

- 当前账号没有学校级角色权限。
- 你没有本场考试的考试内角色。
- 你缺少学生身份字段权限。

处理步骤：

1. 确认你登录了正确组织。
2. 确认你正在访问正确考试。
3. 联系管理员或考试创建人确认角色。
4. 刷新页面或重新登录后再检查入口。

## 答题卡上传失败

可能原因：

- 文件格式不支持。
- 文件过大或损坏。
- 扫描件页码、条码或边角缺失。
- 上传了错误考试的答题卡。

处理步骤：

1. 确认文件为 PDF、JPG 或 PNG。
2. 重新导出或重新扫描文件。
3. 检查条码、页码和答题区域是否完整。
4. 确认答题卡来自当前考试。

## 条码识别失败

处理步骤：

1. 查看原始扫描页。
2. 检查条码是否清晰、完整、未被遮挡。
3. 检查是否上传了旧版本或其他考试答题卡。
4. 必要时重新扫描。
5. 不确定时，不要交付阅卷。

## 学号绑定失败

处理步骤：

1. 进入录入结果。
2. 查看系统识别的学号或异常提示。
3. 确认学生是否属于本场考试考核班级。
4. 人工绑定正确学生。
5. 保存后检查答题卡统计。

## 客观题识别异常

常见异常包括空白、多选、低置信度和识别失败。

处理步骤：

1. 让异常进入客观复核。
2. 阅卷教师查看题目和识别结果。
3. 人工确认答案或分数。
4. 保存后重新计算成绩。

## 阅卷任务为空

可能原因：

- 扫描还没有交付阅卷。
- 阅卷分配还没有完成。
- 当前教师没有被分配到题目。

处理步骤：

1. 检查扫描交付状态。
2. 进入阅卷分配。
3. 确认题目已分配给对应阅卷教师。
4. 让阅卷教师刷新页面。

## 成绩无法发布

可能原因：

- 阅卷任务未完成。
- 成绩尚未重新计算。
- 当前账号没有成绩发布权限。

处理步骤：

1. 检查客观复核和主观阅卷是否完成。
2. 重新计算成绩。
3. 检查是否存在异常学生或缺失答题卡。
4. 联系考试创建人或考官发布成绩。

## 导出字段缺失

如果导出中没有姓名、学号或班级，通常是当前账号没有学生身份字段权限。

处理步骤：

1. 确认当前账号是否应该看到身份字段。
2. 联系管理员调整权限。
3. 重新登录后再导出。

## 完成后你应该看到

- 问题有明确归类。
- 知道应该找考试创建人、学校管理员还是扫描人员处理。
- 不会把权限裁剪误判为系统错误。
